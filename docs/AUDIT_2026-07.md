# Full-system re-audit — July 2026

A fresh adversarial pass over every surface, re-grading from scratch rather
than anchoring on the previous audit. Seven independent auditors, each told to
judge against a world-class bar and to verify claims against the code, not the
docs. The load-bearing findings below were then re-verified by hand
(file:line confirmed) before landing here — none are speculative.

Prior grades in parentheses; this is the second full audit, ~80 workstreams
later.

## Grades

| Surface | Grade | Was | Direction |
|---|---|---|---|
| Reality Compiler v2 / reality-core | **A−** | (new) | — |
| Coverage & CI machinery | **A−** | A− | = |
| Halo (on-glass Lua) | **B+** | A− | ↓ |

| Mac Brain appliance | **C+** | B+ | ↓↓ |

**The through-line:** almost every downgrade is *excellent machinery that
isn't switched on, or isn't exercised on the real path*. The Mac has auth code
that defaults open. The Lens platform has isolation that defaults off. The Halo
has features whose tests call them directly instead of through the live
registration, so two shipped dead. The CI has world-class tests, several of
which no workflow runs. This is a good kind of debt: the hard parts exist and
are well built; the work is wiring, defaulting, and routing tests through the
real path — not new invention.

## P0 — trust-critical, verified, and small

These are correctness/safety bugs on the paths users trust most. Each was
reproduced or confirmed at the cited line.

1. **Mac brain binds the whole LAN with no auth by default.**
   `ai_brain/server/__main__.py` defaults `--host 0.0.0.0` and `--token ""`;
   `_authed()` returns True when the token is empty (`server.py:1925`). On a
   shared network every route is open, and that chains into local-file
   exfiltration (add `/Users` as a folder → reindex → ask the brain to read it
   back). Fix: default host to `127.0.0.1`, auto-generate a token on first run
   if none is set — the sibling `laptop-companion` already models the guard.
2. **Spoken commitments are silently lost, then confirmed as saved.** The
   capture worker runs off-thread; `db.add_commitment` throws sqlite's
   `check_same_thread` error (`memory/db.py:7`, plain `sqlite3.connect`), a
   bare `except: pass` swallows it, and `_capture_commitment` still sends the
   "commitment captured" card (`ops_conversation.py:214-223`). Fix: thread-safe
   DB (a connection lock + `check_same_thread=False`), and only confirm on a
   real write — record failures to the health ledger.
3. **"Erase everything — cannot be undone" is undone by reboot.** Phone
   `purgeAll` clears in-memory state and tells the Brain, but never removes the
   `AsyncStorage` cache key, so `hydrateCache` restores the erased memories at
   next launch (`useMemoryStore.ts:94-101` vs `125-137`). Fix: `purgeAll` also
   removes `CACHE_KEY`.
4. **Two Halo features are dead in production.** `main.lua:48,53` register the
   Timbre and TinCan callbacks without passing `now_ms`, so `until_ms` computes
   as `0 + TTL` and every frame expires after ~2.5 s uptime
   (`dream_renderer.lua:48-54,82-88`). Tests hide it by calling `dr.timbre(0)`
   directly. Fix: thread the tick clock into both registrations.
5. **Figment clamp bypass → display DoS.** A counter named `%` builds a bad
   gsub pattern in `figment_stage.lua:254` and throws on every render →
   permanent crash-guard fallback until banish; `_clamp_ok` never validates
   counter names or line types, so the documented "the stage re-enforces
   clamps at runtime" guarantee isn't honored. Fix: validate/escape slot and
   counter names in the stage.

## P1 — high value, mostly wiring

6. **CI gate topology holes** (the reason several P0 bugs shipped green):
   registry-api Worker tests run in no workflow; `rust-core.yml`'s path filter
   omits `contracts.py`, the Python reference the parity suite proves against,
   and the default pytest job skips parity when the compiled core is absent —
   so a parity break lands green. There is no Python lint/type gate anywhere
   despite shipping `py.typed`. Fix: run the Worker tests, add `contracts.py`
   (and the whole v2 path) to the parity filter, add a ruff+mypy gate.
7. **Incognito/pause recall contract is applied inconsistently.** `ask_brain`
   blocks reads under incognito (`ops_brain_switches.py:22`) while
   `retrace`/`find_way` deliberately allow them, and `recall_conversation`
   skips even the pause gate (`ops_conversation.py:233`) though pause is
   supposed to block both. Some paths reply "Not while you're incognito" where
   the contract says recall is allowed. Fix: one gate helper, applied per the
   pinned semantics (incognito blocks capture/writes not recall; pause blocks
   both), with tests per path.
8. **"Forget" leaves the vector behind.** `RetentionSweep.sweep` and
   `db.purge_memory`/`purge_all` never call `ann.remove` — purged memories stay
   in the `.usearch` index forever: privacy residue plus unbounded growth.
   Fix: route purge/retention through ANN eviction.
9. **Cloud embedder degrades to the 32-d mock per call, silently.** On any
   error `OpenAIEmbeddingProvider` returns the mock vector (`embeddings.py:143`),
   mixing dimensions into the DB where `cosine()` zip-truncates without a
   health record. Fix: fail loud (health ledger) rather than silently swap
   embedding spaces.
10. **The advertised plugin security is largely shelf-ware.** The WASM
    capability host and the TUF trust chain are built and unit-tested but wired
    into no real path; isolation defaults to in-process; every first-party
    package ships unsigned; and the validation gate both executes the plugin's
    `register()` on the reviewer's machine and is AST-bypassable
    (`import os as o; o.system(...)` passed). Fix: sign first-party packages,
    default untrusted plugins to the isolation tier, and stop executing
    unsigned code in the validation gate.
11. **Menu-bar leaks the cloud opt-in.** Turning Incognito off posts
    `cloud_enabled=True` unconditionally (`menubar.py:192`), silently
    re-enabling the now-opt-in-off cloud tier without consent. Fix: restore the
    remembered preference, don't force-on.

## P2 — bigger lifts, real quality ceilings

12. **Interpreter text-length divergence.** The four figment interpreters clamp
    by different units — Python code points, JS UTF-16 units, Lua and Rust
    bytes — and no harness sweeps non-ASCII, the one input class where they
    provably disagree (Rust can split a UTF-8 codepoint; the parity harness
    would raise, not diff). Pick one canonical unit across all four and add a
    non-ASCII + token-in-slot input class to the differential/parity harnesses.
13. **Mutation coverage is one file.** `only_mutate = contracts.py` — the prover
    `budgets.py`, `flash_safety.py`, `vault_sync.py`, `signer.py` have zero
    mutation coverage; the flash gate is near-vacuous (the only flashing element
    is always under the small-area exemption); and one CrossHair "proof"
    (`spend_token`, float-domain) flakes to red under CPU load. Widen mutation
    scope, give the flash analyzer a figment it can actually flag, reclassify
    the flaky proof to the refutation-only set.
14. **Phone shipping hygiene.** Accessibility is near-zero (the one touch
    primitive sets no role/label, so every button is unlabeled to screen
    readers); i18n covers ~20 chrome strings while every screen body is
    hardcoded English; BLE is wired into no caller even on a dev build; demo
    data leaks into real mode after demo-off. Each is a focused lift.
15. **Memory-spine coherence.** Beyond the P0 thread-safety fix, the DB/ANN/
    thread ownership wants one clear owner: `ann.add()` re-saves the whole index
    on every ingest, the `DismissalTracker` shares one 20-event window across
    all card types, and several config knobs are dead
    (`proactive_min_confidence` ignored, `passive_tick_interval_ms` unused).

## What's genuinely excellent (keep and defend)

- **reality-core faithfulness**: a fixed-capacity zero-alloc BSS pool with a
  splitmix64 stream mirrored bit-for-bit in Python; 5,238 swept parity checks
  plus real-Stage scenarios agree line-for-line. The pre-tokenized renderer is
  a genuine security upgrade (slot values are inert data) and is honestly
  flagged as a deliberate divergence, not hidden.
- **The verification layering**: CrossHair → mutmut → 4-language differential
  parity → wasm parity on the safety core, with anti-vacuity guards that prove
  the proofs can actually fail. The DST harness proves itself (plants a race,
  finds it by exploration, replays it by seed) before testing the hub.
- **Failure-honesty as a design system**: the dual-flag PrivacyGate, the
  per-seam health ledger, cloud egress logged-and-counted on every call, the
  phone's connection hysteresis and error-body guards that refuse to wipe local
  memory on a bad response, and Expo Go survivability via disciplined lazy
  native requires.
- **The plugin developer loop actually works** end-to-end first try
  (`new → validate → pack → preview → dev`), the CLI is stdlib-only, and
  hello-lens runs through the real gate in CI so the tutorial can't rot.
- **The Mac's data-egress hygiene**: `cloud_sync.py` strips secrets before any
  egress and encrypts client-side; every sensitive mutation is localhost-gated;
  the panel's XSS discipline held under adversarial reading.

## Suggested sequencing

P0 is one focused pass — five small, high-trust fixes (LAN auth default,
thread-safe capture, purge-clears-cache, the two Halo live-path bugs) — and it
should go first because each is a promise the product currently breaks. P1 is
mostly a CI-and-wiring wave that also prevents recurrence (running the tests
that already exist, closing the parity filter, defaulting isolation on). P2 are
the real ceilings to reach for A-grade across the board.
